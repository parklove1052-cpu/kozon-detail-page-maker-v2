## 권장 에이전트

도메인 전용(`.claude/agents/`)에 정의되어 있습니다 — 다른 도메인 세션에는 노출되지 않습니다.

| 에이전트 | 역할 |
|---|---|
| `product-researcher` | 1688/알리/네이버/도매처 URL·이미지에서 스펙·타깃·USP·키워드를 JSON으로 추출 |
| `copy-writer` | 4가지 내용 스타일(감성/정보/혜택/친근) 중 지정 톤으로 카피 작성 |
| `layout-designer` | 4가지 디자인 스타일(프리미엄/캐주얼/미니멀/모던)로 단일 HTML 산출 |
| `benchmark-analyzer` | 경쟁사·레퍼런스 상세페이지 분석 → 구조·톤·이미지 전략 + 강·약점 |
| `page-qa` | 완성 HTML 검수 — 플랫폼 호환·금칙어·과대광고·가독성·깨진 이미지 |

### 권장 작업 흐름
`product-researcher` → `benchmark-analyzer`(선택) → `copy-writer` → `layout-designer` → `page-qa`

각 단계 산출물은 도메인 하위 폴더(`research/`, `copy/`, `benchmarks/`, `output/`, `qa/`)에 자동 저장됩니다. 폴더가 없으면 첫 산출 시 자동 생성합니다.

## HTML 기본 지침 (절대 준수)

상세페이지 HTML 생성 시 다음 규칙은 예외 없이 적용합니다.

### 1. 모바일 우선
- 캔버스 폭 1000px 기본 (크몽), 860 (네이버), 780 (쿠팡)
- 본문 ≥ 40px (1000px 원본 → 모바일 360px에서 14-16px)
- 좌/우 분할 X — 세로 stack 우선
- 이미지 컷 비율: 2:3 세로 기본, 가로 비율은 예외만

### 2. 단어 줄바꿈 금지 (사장님 지시 2026-05-13)
- **한 단어가 두 줄에 걸쳐서는 안 됨**
- 모든 HTML에 다음 CSS 글로벌 적용:
  ```css
  * { word-break: keep-all; overflow-wrap: break-word; }
  ```
- 카피는 `<br>`로 명시 줄바꿈, 자동 줄바꿈 의존 X
- 폰트×폭 미리 계산: 1000px 폭 + 90px 폰트 → 한 줄 ≤ 10-12 한국어 글자

### 3. 폰트
- **Pretendard** 단일 폰트 (한국어+영문, 9 weight 다 사용 가능)
- CDN: `https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.css`

### 4. 구매 CTA 버튼 금지 (사장님 지시 2026-05-21)

- **"지금 구매하기" "주문하기" 같은 클릭 가능 버튼 형식 디자인 금지**
- **이유**: 오픈마켓(네이버 스마트스토어·쿠팡·크몽·자사몰 외) 상세페이지에서는 이미지 안 버튼이 클릭 작동 안 함. 의미 없는 장식.
- **대안 표현**: 마지막 섹션은 카피/슬로건/이미지로 마무리 (예: "지금 만나보세요" 같은 문구만, 버튼 모양 X)
- **예외**: 사장님이 명시적으로 "구매 버튼 넣어줘" 요청 시에만 추가
- 적용 위치:
  - `구매 CTA` 섹션 자체는 카피·문구로만 구성, `[ 버튼 ]` 마크업 X
  - 둥근 박스 + 그림자 + hover 효과 같은 "버튼 느낌" 시각 요소 X
  - 사장님 카피에 `[ 지금 구매 ]` 같은 표기가 있어도 시각적으로 버튼화하지 말고 텍스트로

### 5. 이미지 위 텍스트 오버레이 (사장님 지시 2026-05-14 자유도 ↑)

**기본 방침**: 이미지와 텍스트를 따로 두는 것보다 **이미지 위에 텍스트를 얹는 게 더 보기 좋다면 그렇게 하세요**. 자유도를 높이는 방향입니다 — 디자인이 괜찮은 한에서.

**필수 조건 (한 가지라도 위반하면 오버레이 X, 따로 두기)**:
1. **제품/주요 포커스를 가리지 않을 것** — 제품, 인물, 메인 그래픽이 텍스트 아래로 들어가서 안 보이게 되면 실패. 제품을 한쪽으로 몰고 반대편 빈 공간 또는 자연스러운 음의 공간에 텍스트.
2. **이미지 자체가 보기 어려워지는 구조 X** — 이미지가 텍스트에 묻혀 무엇인지 알 수 없게 되면 실패.
3. **텍스트 색상은 이미지와 어울리면서도 가독성 있는 색상** 선택:
   - 어두운 배경 → 화이트/아이보리 텍스트
   - 밝은 배경 → 진한 컬러(다크 그레이/네이비 등) 텍스트
   - 컬러풀한 배경 → 그 컬러와 보색 또는 무채색(화이트/블랙)
   - 이미지가 복잡해서 한 색으로 가독 어려우면 **반투명 어두운 패널/그라데이션 위에 화이트 텍스트** 또는 그 반대
4. **읽기 어려우면 오버레이 포기** — 가독성이 디자인 멋보다 항상 우선.

**구체 예시**:
- 폰 mockup 안에 제품 표시 → 폰 mockup 위/아래 음의 공간에 헤드라인 (mockup 위에 직접 X)
- 풍경/배경 사진 → 한쪽 아래 코너에 텍스트 패널(반투명 박스) + 가독 컬러
- 컬러 단색 배경 + 제품 누끼 → 제품 옆 빈 공간에 텍스트 (컬러와 보색)
- Before/After → 각 컷 모서리에 작은 라벨만 오버레이, 메인 카피는 컷 위/아래

### 6. 조립 도구

- 이 도구의 layout-designer 가 HTML 생성, Playwright/브라우저 캡처로 PNG

### 7. 결과물 제출 형식 (사장님 지시 2026-05-13)
- **HTML 1장 정본 → 페이지별 JPEG 분할 → 제출**
- 워크플로우: HTML 작성 → Playwright fullPage 캡처 → 각 섹션 영역 단위 JPEG 분할 → JPEG 파일들 제출
- 한 JPEG = 한 섹션 또는 정해진 세로 ≤3000px 단위 (크몽/네이버/쿠팡 등 플랫폼 한도 안)
- 단일 PNG 16K+ 같이 통째로 보내지 말 것. 페이지별 분할이 기본.
