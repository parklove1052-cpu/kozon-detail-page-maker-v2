# 구매 CTA 버튼 디자인 금지

## 규칙
상세페이지 마지막에 **"지금 구매하기" "주문하기" 같은 버튼 형식의 디자인을 만들지 않습니다.** 사장님이 명시적으로 요청한 경우에만 추가.

**Why**: 사장님이 만드는 상세페이지는 **오픈마켓(네이버 스마트스토어·쿠팡·크몽·자사몰 외)** 판매용. 이미지 본문 안의 "버튼"은 클릭 작동 안 함 — 의미 없는 장식이고 가독성·신뢰도만 떨어뜨림. 실제 구매 행위는 플랫폼 자체의 상단/하단 구매 버튼에서 일어남.

**How to apply**:
- `구매 CTA` 섹션 자체는 카피·슬로건·이미지로만 구성
- `<button>` 마크업, 둥근 박스 + 그림자 + hover 효과 같은 "버튼 느낌" 시각 요소 모두 X
- 사장님 카피에 `[ 지금 구매 ]` 같은 표기가 있어도 텍스트 한 줄로만 표시 (버튼 모양 X)
- 마지막 섹션은 잔잔한 카피·풀샷 이미지로 마무리하거나, 단순 안내 문구만 표시
- **예외**: 사장님이 "구매 버튼 넣어줘" 또는 "CTA 버튼 추가해줘" 명시 요청 시에만 추가

## 적용 위치
- `domains/상세페이지 제작자/CLAUDE.md` — HTML 기본 지침 4번
- `domains/상세페이지 제작자/code/server.js` — buildGeneratePrompt 출력 규칙

## 사장님 직접 지시
2026-05-21 — "상세페이지 만들 때, 항상 지금 구매하기 버튼이 마지막에 있던데, 그건 빼는게 좋겠어. 우린 오픈마켓 판매 상세페이지 제작이라 마지막에 그런 버튼 형식은 어차피 작동 안 하거든. 이거 기본 지침으로 만들어놔. 따로 요청하지 않을 시에 버튼 형식의 디자인은 구성 x"
