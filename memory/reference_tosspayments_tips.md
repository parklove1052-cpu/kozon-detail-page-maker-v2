# 상세페이지 작성 노하우 3가지

**출처**: 토스페이먼츠 블로그 「바로 써먹는 쇼핑몰 상세페이지 작성 노하우 3가지」(2023-05-04)
**저장**: 2026-05-16 사장님 지시
**용도**: copy-writer / layout-designer 에이전트가 카피·섹션 구조 결정 시 체크리스트로 활용

---

## 1️⃣ 숫자 활용

### 1-A. 장점은 줄글 대신 번호로
- 같은 내용이라도 `1. 2. 3.` 로 끊으면 한눈에 들어옴
- 패턴: **숫자 + 소제목 + 설명 + 관련 이미지**
- **최대 3~4개**까지만 — 너무 많으면 다시 줄글처럼 안 읽힘

### 1-B. 추상적 표현 X, 구체 수치 O
- ❌ "많이 팔린 상품", "인기 상품"
- ✅ "이번 달 5,200개 판매", "리뷰 2,300개", "카테고리 판매 1위"
- 매출액·리뷰 수·순위 = **타인이 품질을 증명한 사회적 증거**
- **수치 기준도 같이 명시** (날짜/카테고리) — 변할 수 있으니까
  - 예: "2022년 6월 기준 침대 카테고리 리뷰 수 1위"

---

## 2️⃣ 스토리텔링 (고객 문제 → 우리 상품)

### 2-A. 고객의 불편함 → 공감 → 해결
- 상세페이지 첫 부분: **"혹시 이런 상황 불편하지 않으셨나요?"**
- 그 뒤 **"이러한 상황을 해결하기 위해 ○○ 상품을 만들었습니다"**
- 예: 거품 새치 샴푸 — 기존 새치 샴푸의 흘러내림·번거로움 → 거품형으로 해결

### 2-B. 고객이 몰랐던 문제 일깨우기
- 평소엔 문제로 인식 안 하던 상황도 문제로 정의 가능
- 예: 수돗물에 만족하던 고객 → 수돗물 잔류 물질 일깨움 → 수도꼭지 필터 제시

### 2-C. 구매 망설임 미리 해결
- 구매 직전 떠오를 의심·고민을 선제적으로 답해줌
- 활용 섹션: **자주 묻는 질문 / 활용법 / 꿀조합**
- 예: 파김치
  - "남은 양념 처분 귀찮은데…" → 양념 활용법 소개
  - "같이 먹을 게 없는데…" → 함께 먹으면 좋은 꿀조합
  - "너무 매우면 어쩌지…" → 매운 정도 안내
- 망설일 이유가 사라지면 → 안 살 이유가 없어짐

---

## 3️⃣ 리뷰 활용

### 3-A. 실제 리뷰를 상세페이지에 노출
- 좋은 리뷰가 묻혀 있으면 의미 X → 본문에 가져와 박기
- **정보가 있는 리뷰** 선별 (사이즈·핏·임산부 사용감·키 매칭 등)
- 클릭 시 실제 리뷰 페이지로 연결 → 신뢰도 ↑

### 3-B. 사용 전/후 비교
- 눈에 보이는 변화가 있는 상품은 **Before/After** 무조건
- 고객이 직접 찍은 사진이면 신뢰도 폭발
- 초기 판매라 리뷰 부족 시: **체험단·리뷰 이벤트** 활용

## 💪 3줄 핵심 요약 (체크리스트)

- [ ] 상품 설명에 **구체적인 숫자**(매출/리뷰/순위 + 기준)가 들어가 있는가?
- [ ] **고객의 문제** → **공감** → **우리 상품이 해결** 흐름이 있는가?
- [ ] **실제 리뷰**(특히 정보 담긴 리뷰 + 사용 전/후)가 상세페이지에 노출되는가?

## 자주 쓸 카피 패턴 모음 (재사용)

- "혹시 이런 상황 불편하지 않으셨나요?"
- "이러한 상황을 해결하기 위해 ○○을 만들었습니다."
- "○○월 기준 판매량 N개" / "리뷰 N개" / "카테고리 N위"
- "함께 먹으면 좋은 꿀조합 / 활용법"
- "자주 묻는 질문 — 사기 전 고민, 미리 해결해드릴게요"
- Before / After 박스

## 적용 메모

- product-researcher: 사장님 상품에서 **수치화 가능 지표** 적극 추출 (판매수·리뷰수·순위·인증·OEM 경력 등)
- copy-writer: 헤드라인·서브카피 작성 시 위 카피 패턴 적극 활용. 추상 표현 자동 차단
- layout-designer: USP 섹션은 1·2·3 번호로 끊기 (최대 4개). FAQ·활용법 섹션 빠뜨리지 말 것. 리뷰 인용 박스 + Before/After 박스 템플릿 보유
- page-qa: 체크리스트 3개로 검수
