## 만약 또 실패하면 우리가 다음에 시도할 것 (다음 라운드 후보)

### 1순위 — 첨부 이미지 두 번 전송 줄이기
현재: ⚡에서 plan과 generate 양쪽에 base64 이미지 전송 → 큰 이미지면 payload 2배.
해결: plan 응답에서 `session_dir` 받아 generate에는 session_dir만 전달, 서버가 그 폴더의 파일을 재사용.
- handlePlan은 이미 session_dir 반환 중 (reference_session_dir 필드)
- handleGenerate가 session_dir 받아 이미지 재사용하도록 변경 필요
- app.js callDirectGenerate 2단계에서 base64 대신 session_dir + 메타만 전송

### 2순위 — 백그라운드 job + 폴링
진짜 5분+ 작업이면 클라 fetch 자체를 비동기로:
- POST /api/generate → 즉시 jobId 반환 (202 Accepted)
- 서버는 백그라운드로 claude 호출
- 클라는 /api/job/<id> 1~3초마다 폴링
- 완료되면 결과 다운로드
장점: 클라 timeout 완전 제거, 페이지 닫아도 보존, 진행률 표시
단점: 구조 변경 큼

### 3순위 — Codex 1라운드 미반영 항목
- normalizePath 한계 (백슬래시·대소문자만) → 첨부 이미지 short id 발급으로 경로 매칭 대체
- disabled 버튼 접근성 → aria-describedby 패턴

## 핵심 알림 (사장님이 자주 헷갈리시는 점)
1. **서버 재시작 안 하면 옛 코드** — 어제 사장님이 timeout 180초로 끊긴 게 이것 때문. config.json 360초로 바꿔도 안 반영. 이번엔 callClaude 핫 리로드라 timeout만큼은 재시작 없이 반영되지만, **그 외 라우트·prompt 빌드는 재시작 필요**.
2. **브라우저 캐시** — Ctrl+F5 안 누르면 옛 app.js가 동작. 새 ⚡ 2단계 흐름이 안 나옴.
3. **uploads/output 폴더** — 시간 지나며 쌓임. 가끔 청소 필요 (디스크 점유).
