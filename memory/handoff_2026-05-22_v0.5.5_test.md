## 핵심 도구 사용 흐름 (다음 세션 참고)

### Step 1 — 이미지 첨부
- 좌(제품 사진) / 우(기타 이미지) 2칸 dropzone
- 각 이미지 썸네일 밑 **이미지 설명 (선택)** input — 사장님 적은 설명이 LLM prompt에 들어감
- 클립보드 paste는 마지막 hover/클릭한 dropzone

### Step 1 끝 — 두 갈래 버튼
- ① **기획 + 이미지 프롬프트 생성** (이미지 ChatGPT로 받을 때)
- ⚡ **주어진 자료로 바로 제작** (이미지 다 있을 때, plan → generate 자동 연쇄)

### Step 4 — 결과 다운로드 (Step 잠금 무관 상시 노출)
- 📄 상세페이지 HTML 다운받기 (즉시 blob)
- 🖼️ JPEG로 다운받기 (job → download_url 폴링)

## 실패 시 진단 자산

- 다운로드 영역 색상: 🟦 idle → 🟪 pending → 🟩 success → 🟥 failed
- 빨간 에러 박스 영구 표시: 어느 단계 실패인지 + 서버 정보 자동 노출
- 옛 서버 감지 시 즉시 경고 토스트 + 에러 박스에 🚨 안내 + `taskkill /F /PID` 명령어
- 브라우저 콘솔 (F12): `[direct-generate]` `[generate:plan]` `[pollJob]` 로그
- 서버 콘솔: `[plan|generate|extract|render-jpeg <jobId>]` 로그
- 디스크 디버그 파일: `code/uploads/_last_prompt.txt`, `_last_output.txt`
